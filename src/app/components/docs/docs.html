Project Blueprint & Google Cloud Architecture

01. System Architecture 

Layer 1: Frontend (UX/UI)
   [Angular App] --- (Firebase Auth) --- [Google/Guest]
          |
Layer 2: Storage & Real-time Data
   [Cloud Firestore] <--- (Real-time Listeners) ---> [AngularFire]
          |
Layer 3: Serverless Logic
   [Firebase Functions] <--- (OnWrite Triggers) --- [Firestore]
          |
Layer 4: AI & Compute
   [Cloud Run Proxy] <--- (JSON API) ---> [Gemini 1.5 Flash]
          |
Layer 5: Messaging & Scheduling
   [Cloud Scheduler] ---> [Pub/Sub] ---> [Functions (Streak Reset)]
          |
Layer 6: Analytics & Insights
   [BigQuery] <--- (Firebase Export) --- [Firebase Analytics]

02. Firestore Schema

/users/{uid}

Core user profile and gamification data.

xp: number // Total earned XP

level: number // Calculated via XP waterfall

streak: number // Consecutive active days

badges: string[] // Earned badge IDs

/predictions/{id}

User-locked predictions for match outcomes.

matchId: string

winner: string

scoreRange: [number, number]

status: 'pending' | 'processed'

03. Security Rules 

rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /users/{userId} {
      allow read: if request.auth != null;
      allow write: if request.auth.uid == userId;
    }
    match /leaderboard/{doc} {
      allow read: if true;
      allow write: if request.auth.token.admin == true;
    }
  }
}
